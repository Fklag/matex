La division euclidienne

  • Le dividende étant la quantité à partager.
  • Le diviseur étant le nombre de parts ; non nul.
  • On souhaite un quotient entier et un reste.

On a bien b≠0b \neq 0b​=0, et :

a÷b→(q,r)a\div b \rightarrow (q, r)a÷b→(q,r),
où a=b×q+ra=b\times q + ra=b×q+r, avec 0⩽r<b0\leqslant r<b0⩽r<b.

aaa est le dividende, bbb est le diviseur.
qqq est le quotient, rrr est le reste.

On parle de division modulo bbb, (divmod en abrégé).

Exemple
410÷30→410\div 30 \rightarrow410÷30→ quotient =13= 13=13, reste =20= 20=20,
on a bien 410=30×13+20410 = 30\times 13 + 20410=30×13+20, et 0⩽20<300 \leqslant 20 < 300⩽20<30.

On dit que 410410410 modulo 303030 est égal à 202020.

⚠️ Attention
410÷13→410\div 13 \rightarrow410÷13→ quotient =30= 30=30 , reste =20= 20=20 ; est faux,
on a bien 410=13×30+20410 = 13\times 30 + 20410=13×30+20, mais pas 0⩽20<130 \leqslant 20 < 130⩽20<13.

Test avec Python :

>>> 13*30 + 20 == 410
True
>>> 20 < 30
True

Avec Python3, le quotient entier de aaa par bbb s'obtient avec a // b,
et le reste dans cette division s'obtient avec a % b.
On peut aussi obtenir le quotient et le reste simultanément avec divmod(a, b).

Exemple, avec la situation précédente :

>>> 410 // 30

>>> 410 % 30

>>> divmod(410, 30)
(13, 20)
>>> divmod(410, 13)
(31, 7)

Factorielle d'un entier

Définition
On note n!=1×2×3×...×nn! = 1×2×3×...×nn!=1×2×3×...×n, la factorielle de nnn.
Exemple
5!=1×2×3×4×5=1205! = 1×2×3×4×5 = 1205!=1×2×3×4×5=120

Avec Python, on peut la calculer en important une fonction incluse dans le module math :

>>> from math import factorial
>>> factorial(5)
120

Attention, sur la calculatrice NumWorks, la version de Python n'inclut pas factorial, on peut alors créer le script suivant :

# Version simple
def factorial(n):
    ans = 1
    for x in range(1, n+1):
        ans = ans * x
    return ans

# Version NSI, avec la récursivité
def factorial(n):
    """ Renvoie la factorielle de n
    n doit être un entier positif.
    Version récursive pour les NSI
    >>> factorial(0)
    1
    >>> factorial(5)
    120
    """
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

III] Liste des diviseurs

On rappelle que aaa, bbb et nnn sont des entiers non nuls.

Propriété : si aaa est un diviseur de nnn, alors na\dfrac{n}{a}an​ est entier et aussi un diviseur de nnn.

Preuve : Si aaa est un diviseur de nnn, alors ∃ b∈N∗\exists\ b \in \mathbb{N}^*∃ b∈N∗ tel que a×b=na \times b = na×b=n, ainsi b=nab = \dfrac{n}{a}b=an​ est entier et un diviseur de nnn.

Propriété : Si a×b=na\times b=na×b=n, avec a<ba < ba<b, alors a<na < \sqrt{n}a<n​.

Preuve 1 : a×a<a×b=na\times a < a\times b = na×a<a×b=n, avec ⋅\sqrt{\cdot}⋅​ croissante sur R+\mathbb{R}^+R+, on a: a×a<n\sqrt{a×a} < \sqrt{n}a×a​<n​, donc a<na<\sqrt{n}a<n​.

Preuve 2 : Si n⩽a<b\sqrt{n} \leqslant a < bn​⩽a<b, alors a×b>(n)2=na\times b > (\sqrt{n})^2 = na×b>(n​)2=n ; absurde.

Utilisation : Pour obtenir la liste des diviseurs de nnn non nul, il suffit de trouver ceux inférieurs à n\sqrt{n}n​, d'y adjoindre leur 'binôme', et de tester éventuellement n\sqrt{n}n​.

On va tester tous les diviseurs de 111 jusqu'à 181818. En effet, 18<345<1918<\sqrt{345} < 1918<345​<19.
345=1×345345 = 1\times 345345=1×345
345=3×115345 = 3\times 115345=3×115
345=5×69345 = 5\times 69345=5×69
345=15×23345 = 15\times 23345=15×23
La liste des diviseurs de 345345345 est {1,3,5,15,23,69,115,345}\{1, 3, 5, 15, 23, 69, 115, 345\}{1,3,5,15,23,69,115,345} , il y en a 888.

Il faut ne compter qu'une fois les diviseurs d'un carré, comme 36…36 \dots36…

On va tester tous les diviseurs de 111 jusqu'à 36=6\sqrt{36} = 636​=6.

IV] Nombres premiers

Pour tout entier n>1n>1n>1, on a 1×n=n1\times n = n1×n=n, donc nnn possède au moins deux diviseurs : 111 et lui-même.

Définition : Un nombre premier est un entier qui a exactement deux diviseurs : 111 et lui-même.

Les nombres premiers inférieurs à 202020 sont : 2,3,5,7,11,13,17,192, 3, 5, 7, 11, 13, 17, 192,3,5,7,11,13,17,19.

Remarque : Un entier n>1n>1n>1 est soit premier, soit composé.
Son plus petit diviseur, excepté 111, est premier.

111 est à part, il n'est ni premier, ni composé.

Théorème : Il y a une infinité de nombres premiers.

Preuve 1 : Soit n∈N∗n \in \mathbb{N}^*n∈N∗, on considère N=1+n!N = 1 + n!N=1+n!, qui n'a aucun diviseur entre 222 et nnn, donc son plus petit diviseur est supérieur à nnn, il est premier. On déduit qu'il y a des nombres premiers aussi grands que l'on veut, une infinité donc.

Preuve 2 : Par l'absurde, supposons qu'il y ait un nombre fini de nombres premiers notés pip_ipi​ (pour iii de 111 à kkk). On considère N=1+p1×p2×…pi⋯×pkN = 1+p_1\times p_2\times \dots p_i \dots\times p_kN=1+p1​×p2​×…pi​⋯×pk​. On sait que 222 est premier, donc k>0k>0k>0 et N>1N>1N>1 qui possède alors un plus petit diviseur (hors 111) qui est un nombre premier pjp_jpj​. Cependant le reste de la division de NNN par pjp_jpj​ est 111 qui n'est donc pas un diviseur. Contradiction.

def is_prime(n):
    "Renvoie True si n est premier, False sinon"
    if n < 2:
        return False
    for d in range(2, n):
        if n % d == 0:
            return False
    return True

Que l'on teste :

>>> [n for n in range(20) if is_prime(n)]
[2, 3, 5, 7, 11, 13, 17, 19]

Cette méthode est lente avec des nombres nnn trop grands, (comme un milliard ou plus).
Dans le pire des cas, la boucle de is_prime fait presque nnn tours, donc un nombre d'opérations proportionnel à nnn. On dit alors que sa compléxité temporelle est en O(n)\mathcal{O}(n)O(n).

def is_prime(n):
    "Version améliorée"
    if n < 2:
        return False
    for d in range(2, n):
        if n % d == 0:
            return False
        if d * d > n:
            return True
    return True

>>> [n for n in range(10**9, 10**9+100) if is_prime(n)]
1000000007, 1000000009, 1000000021, 1000000033, 1000000087, 1000000093, 1000000097,

Cette méthode fait, dans le pire des cas, n\sqrt{n}n​ tours de boucle pour savoir si nnn est premier. On dit alors que sa complexité temporelle est en O(n)\mathcal{O}(\sqrt{n})O(n​).

Elle sera lente pour des entiers premiers de 181818 chiffres ou plus, ou composés avec le plus petit facteur premier à 999 chiffres ou plus. Dans ces cas, elle fera plus d'un milliard d'opérations, ce qui n'est pas instantané. Elle restera rapide pour des nombres très grands qui possèdent un petit facteur premier.

V] PGCD, PPCM

On rappelle que aaa, bbb et nnn sont des entiers non nuls.

On note D(a)\mathcal{D}(a)D(a) l'ensemble des diviseurs de aaa, c'est un ensemble non vide et fini, aaa étant le plus grand élément.

Propriété : 111 appartient à D(a)\mathcal{D}(a)D(a) et D(b)\mathcal{D}(b)D(b), ainsi leur intersection est non vide, qui est de plus une partie finie. Cette intersection possède donc un plus grand élément.

Définitions :

  • PGCD(a,b)\textrm{PGCD}(a, b)PGCD(a,b) est le plus grand élément de D(a)∩D(b)\mathcal{D}(a) \cap \mathcal{D}(b)D(a)∩D(b).
    C'est le Plus Grand Commun Diviseur, (gcd : greatest common divisor ; in english).

  • PPCM(a,b)\textrm{PPCM}(a, b)PPCM(a,b) est le Plus Petit Commun Multiple à aaa et bbb, (lcm : least common multiple ; in english). C'est un entier inférieur ou égal à ababab qui est un multiple commun trivial.

Exemple 1
D(9)={1,3,9}\mathcal{D}(9) = \{1, 3, 9\}D(9)={1,3,9} et
D(14)={1,2,7,14}\mathcal{D}(14) = \{1, 2, 7, 14\}D(14)={1,2,7,14}, donc
D(9)∩D(14)={1}\mathcal{D}(9) \cap \mathcal{D}(14) = \{1\}D(9)∩D(14)={1}.
Enfin PGCD(9,14)=1\textrm{PGCD}(9, 14) = 1PGCD(9,14)=1.

Exemple 2
Les multiples de 121212 sont : 12,24,36,48,60,72…12, 24, 36, 48, 60, 72\dots12,24,36,48,60,72…
Les multiples de 151515 sont : 15,30,45,60,75,…15, 30, 45, 60, 75, \dots15,30,45,60,75,…

Ainsi PPCM(12,15)=60\textrm{PPCM}(12, 15) = 60PPCM(12,15)=60.
